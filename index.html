The Riku Operating System

The Riku Operating System is a home-made, hobby operating system for Intel x64 architectures.

It is divided into two main components :

  • loader, which is the x86 bootstrap code,
  • kern, which is the x64 kernel code.

Right now, Riku isn't ready for an everyday use, or even a real-hardware boot. This should happen in the future, or at least I hope it does!

Required stuff

In order to compile and run Riku, some standard development environment is required :

  • x86_64-elf-gcc toolchain (mandatory)
  • grub2 (grub-mkrescue to be honest, to generate the riku.iso bootable image)
  • nasm (my favourite assembler)
  • qemu-system-x86_64 (a virtual machine to run the kernel on is always a good thing)
  • GNU make (yep)
  • Tons of coffee (don't tell me you don't need this, I won't believe you)

Also, I know the Makefile are still a bit crappy, but I'm too lazy to do a clean thing. Feel free to make it cleaner by contributing!

What is done

Right now, Riku can initialize some CPU-related components, as well as a graphical terminal (using QEMU's BGA adapter) :

  • x86 to x64 switch
  • Segment descriptors (GDT)
  • Interrupt management (IDT)
  • MMU configuration
  • Heap management

What's next

These are a few functionalities which should be added in the few next commits :

  • Threading (kernel tasks)
  • Driver model
  • Hardware probe and driver initialization
  • Virtual filesystem
  • System call interface
  • Userland binaries
  • Real filesystem
  • And a lot more...

Some words about Riku

Keep in mind that Riku is a hobby kernel, which means its development cycle is slow, and that the source code might be buggy as hell. This is a personal project with educational purposes, so even if I hope I'll finish it one day, it will probably never be as stable and usable as a commercial OS.

That being said, don't hesitate to contribute to the project, that would really be helpful!
